# Item-Specific Context Policy

AIDA no longer creates learner-facing example sentences by dropping arbitrary vocabulary into a small set of reusable templates.

## Sentence rule

A sentence may appear only when it is tied to the exact vocabulary or grammar item through one of these routes:

1. a manually audited example,
2. an item-specific curated example,
3. an exact-match corpus sentence with an English translation, or
4. a bundled source example that contains the exact target form.

If an item has no trustworthy example, the interface shows no sentence rather than fabricating filler.

## Grammar rule

Japanese grammar uses a per-item registry. High-risk points use manual audits. Other points may use authentic corpus examples only when a strong literal construction signature occurs in the sentence. Character overlap alone is not accepted as grammatical evidence.

Cantonese grammar uses the manually audited per-item override bank. Runtime placeholder substitution has been removed.

## Multi-sentence rule

Independent sentence examples are never joined together to make a passage. Multi-sentence study material comes only from the curated sentence/passage banks, where the later sentence belongs to the same situation as the earlier one. Context-browser passage matches must contain the exact vocabulary form.

## Everyday vocabulary

The V12 everyday supplement now has item-specific examples for all 83 added Japanese items and all 49 added Cantonese items. These examples describe plausible uses or situations for the exact item rather than reusing a generic “I bought X” or “X is important” frame.
